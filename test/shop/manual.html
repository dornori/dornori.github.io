Introduction

Overview

The LUMIO shop is a fully self-contained HTML/CSS/JavaScript webshop. There is no backend, no build step, and no package manager. Drop the folder onto any static web host (Netlify, Vercel, GitHub Pages, plain hosting) and it works immediately.

All configuration lives in three places:

FileWhat it controls
js/config.jsShop name, payment keys, tax rate, feature flags, file paths
data/products.jsonAll products — their prices, images, videos, variants, bundles
data/shipping.csvPer-country shipping rates, free-shipping thresholds, delivery times
data/currencies.csvExchange rates, symbols, decimal formatting per currency
data/lang/ui/en.jsonAll UI text labels (supports en, de, nl, no)
data/lang/products/en.jsonTranslated product names and descriptions

💡

The data/products/ folder with individual <product>.json files is not read by the app. The app reads only data/products.json. The individual files are documentation references only.

Getting Started

Pages & URLs

index.html — The Shop Grid

Displays all products as a browseable grid with category filter tabs. No URL parameters needed.

index.html

product.html — Product Detail Page

Shows a full product page with gallery, variants, add-to-cart, bundle section, specs accordion and related products. By default it loads the arc-floor-lamp, but any product can be loaded by passing its ID in the URL:

product.html?id=concrete-desk-lamp

The id must exactly match the product's "id" field in products.json. If no match is found, the first product in the list is shown as a fallback.

product1.html — Second Product Page

Identical to product.html but defaults to the concrete-desk-lamp. Useful when you want two different default products on two different linked pages. Same ?id= parameter works here too.

cart.html — Checkout

Three-step checkout: Shopping Cart → Customer Details → Review & Pay. Cart contents are stored in localStorage so they persist between page visits. No URL parameters.

success.html — Order Confirmed

Shown after a successful PayPal or Stripe payment. Displays the order reference number and clears the cart.

ℹ️

Tip: You can link directly to any product from anywhere on your site using product.html?id=your-product-id. This is how the "Related Products" and "Frequently Bought Together" sections generate their links.

Products

products.json

Located at data/products.json, this is a JSON array where each object is one product. The order of products in this array is the order they appear in the shop grid.

// data/products.json — structure overview
[
  { /* product 1 */ },
  { /* product 2 */ },
  ...
]

A complete product entry looks like this:

{
  "id":             "arc-floor-lamp",        // unique slug, used in URLs
  "price":          249,                     // base price in EUR
  "weight":         4.2,                     // in kg, used for shipping calc
  "stock":          12,                      // total stock (if no variants)
  "category":       "floor",                 // used for filter tabs
  "featured":       true,                    // shows "Featured" badge on card
  "image":          "https://...",           // fallback / card image
  "images":         ["https://...", ...],     // gallery images (first = main)
  "videos":         ["https://..."],          // YouTube embed URL or mp4 URL
  "url":            "product.html",          // link from card/grid
  "variants":       [...],                   // size/finish options
  "colors":         ["Smoke", "Amber"],      // colour swatch names
  "colors_soldout": ["Amber"],              // which colours are sold out
  "bundled":        ["dimmer-switch"],        // "Frequently Bought Together"
  "bundleDiscount": 10,                      // % discount when buying bundle
  "related":        ["globe-pendant"],        // "Related Products" section
  "addons":         ["lamp-cable-3m"],        // accessory strip (future use)
  "specs":          [...],                   // accordion spec sections
  "dimensions":     { "l":40, "w":40, "h":180 }  // in cm
}
Products

All Product Fields

FieldTypeRequiredDescription
idstringrequiredUnique slug. Used in ?id= URLs, bundle references, related lists. Lowercase, hyphens only. Example: "arc-floor-lamp"
pricenumberrequiredBase price in EUR (the shop's internal currency). All other currencies are calculated from this.
weightnumberoptionalWeight in kilograms. Used for shipping cost calculation. Defaults to 0 if omitted.
stocknumberoptionalAvailable stock units. Set to 0 to mark as sold out. Only used when the product has no variants. Defaults to 99.
categorystringoptionalCategory slug shown in the filter bar. Examples: "floor", "pendant", "desk", "wall", "accessory". Add any new value and a filter button appears automatically.
featuredbooleanoptionalSet true to show a "Featured" badge on the product card.
imagestringrequiredPrimary image URL. Shown on product cards. Can be a full URL or a relative path.
imagesarrayoptionalArray of image URLs for the product page gallery. First image = main. All images appear as clickable thumbnails. Supports Unsplash ?w=800 sizing params.
videosarrayoptionalArray of video URLs. Supports YouTube embed URLs and direct mp4 URLs. See Images & Videos.
urlstringoptionalURL the product card links to. Typically "product.html" or "product.html?id=your-id". Leave empty ("") for products with no dedicated page (accessories).
variantsarrayoptionalSize/finish/option variants. Each has its own price, stock and image. See Variants.
colorsarrayoptionalColour swatch names. Shown as circular buttons. See Color Swatches.
colors_soldoutarrayoptionalList of colour names from colors that are sold out. They remain visible but are disabled with a strikethrough.
bundledarrayoptionalArray of product IDs shown in the "Frequently Bought Together" section on the product page. See Bundles.
bundleDiscountnumberoptionalPercentage discount applied when all bundled items are added together. Example: 10 = 10% off the combined price.
relatedarrayoptionalArray of product IDs shown in the "Related Products" section below the main product info.
addonsarrayoptionalArray of product IDs for accessory/addon suggestions. Reserved for future use.
specsarrayoptionalAccordion specification sections on the product page. See Specs.
dimensionsobjectoptionalProduct dimensions: { "l": 40, "w": 40, "h": 180 } in centimetres. Shown in the product info widget.
Products

Variants

Variants are different options of the same product — finish, size, material. Each variant can have its own price, stock count and image. When variants are present, the product-level stock field is ignored.

"variants": [
  {
    "id":    "matte-black",    // unique within this product
    "label": "Matte Black",   // shown on the button
    "price": 249,             // price in EUR
    "stock": 8,               // 0 = sold out, shown disabled
    "image": "https://..."    // main image when this variant is selected
  },
  {
    "id":    "brushed-gold",
    "label": "Brushed Gold",
    "price": 279,
    "stock": 4,
    "image": "https://..."
  },
  {
    "id":    "obsidian-black",
    "label": "Obsidian Black",
    "price": 99,
    "stock": 0,              // ← stock:0 marks it sold out
    "image": "https://..."
  }
]
💡

Set "stock": 0 on a variant to mark it sold out. It remains visible with a strikethrough label so customers know the option exists. Do not remove it from the array.

Products

Color Swatches

Colour swatches are for products where the only difference is colour — no price difference, no separate images via the variant system. They are shown as small circular buttons.

"colors":         ["Smoke", "Clear", "Amber"],
"colors_soldout": ["Amber"]   // Amber shows with strikethrough, unclickable

The swatch background colour is set in the SWATCH_COLORS map inside buy-now-example.html and also in the shop.css nth-child fallback rules. Currently defined colours:

NameHex
Smoke#a0968c
Clear#c0d0e0
Amber#d0a040
Opal White#e8e4dc
Terracotta#c87050
Matte Black#2a2825
Brushed Gold#d4a83c
Raw Concrete#b0a898
Washed White#f0ece4
Obsidian Black#1a1714

To add a new colour, add its name to both the product's colors array and the SWATCH_COLORS object in buy-now-example.html. For the main shop, also add an nth-child rule in shop.css or the colour will render as white.

⚠️

Do not use both variants and colors on the same product. If both are present, variants take priority and colors are ignored.

Products

Images & Videos

Images

The image field (singular) is the card/fallback image. The images array is for the product page gallery — all images appear as clickable thumbnails, and the first image is shown in the main slot by default.

"image":  "https://images.unsplash.com/photo-xxx?w=800&q=80",
"images": [
  "https://images.unsplash.com/photo-xxx?w=800&q=80",   // main/first
  "https://images.unsplash.com/photo-yyy?w=800&q=80",   // second thumb
  "https://images.unsplash.com/photo-zzz?w=800&q=80"    // third thumb
]

Images can be any URL — Unsplash, your own CDN, or relative paths like "images/my-lamp.jpg".

Videos

The videos array supports two formats:

TypeFormatExample
YouTubeUse the /embed/ URL format"https://www.youtube.com/embed/VIDEO_ID"
MP4 fileAny direct .mp4 URL"https://yoursite.com/video.mp4"
 
"videos": [
  "https://www.youtube.com/embed/9bZkp7q19f0",   // YouTube
  "https://yoursite.com/lamp-demo.mp4"            // MP4
]

On the product page, videos appear as thumbnails in the image strip with a ▶ play icon. YouTube thumbnails are automatically fetched from YouTube. Clicking a video thumb loads it inline in the main image area. Clicking the main image area opens the lightbox — if a video is playing it opens directly to the Video tab.

⚠️

For YouTube, always use the embed URL format: youtube.com/embed/VIDEO_ID, not the regular watch URL (youtube.com/watch?v=). The embed URL is required for the iframe to work.

For MP4 files, the server hosting the video must allow cross-origin requests (CORS). Videos hosted on the same domain as the shop always work. External CDNs vary.

Products

Bundles & Related Products

Frequently Bought Together

The bundled array lists product IDs shown in the "Frequently Bought Together" section on the product page. A single "Add Bundle to Cart" button adds all items at once. If bundleDiscount is set, the combined price is shown with the discount applied and a 🏷️ icon.

"bundled":        ["dimmer-switch", "replacement-bulb-e27"],
"bundleDiscount": 10   // 10% off the combined price of all items

If bundleDiscount is 0 or omitted, the total price is shown without a discount.

Related Products

The related array lists product IDs shown in the "Products Related to This Item" section. These appear as cards with a "Buy Now" hover button.

"related": ["globe-pendant", "half-moon-sconce", "concrete-desk-lamp"]
💡

Both bundled and related reference products by their id. The referenced product must exist in products.json. If a referenced ID is not found it is silently skipped.

Specs & Dimensions

Spec Accordion

The specs array creates collapsible accordion sections on the product page. Each section has a title and a table of key-value rows.

"specs": [
  {
    "title_key": "specs_features",  // key for translated title (optional)
    "title":     "Features & Specs", // fallback title if key not found
    "rows": [
      ["Light Source", "LED"],
      ["Bulb Base",     "E27"],
      ["Dimmable",      "Yes"],
      ["Max Wattage",   "40W"]
    ]
  },
  {
    "title": "Measurements",
    "rows": [
      ["Height", "180 cm"],
      ["Weight", "4.2 kg"]
    ]
  }
]

Dimensions

The dimensions object is separate from specs and is used in the product info widget to show L × W × H cm.

"dimensions": { "l": 40, "w": 40, "h": 180 }  // all in cm
Products

Stock & Sold Out

There are three ways to mark something as sold out depending on the product type:

SituationHow to mark sold out
Whole product (no variants)Set "stock": 0 on the product
A specific variantSet "stock": 0 on that variant object
A specific colour swatchAdd the colour name to "colors_soldout"

In all cases the option remains visible so customers can see it exists, but it is rendered disabled with a strikethrough and cannot be selected or added to cart.

// Sold out whole product
{ "id": "my-lamp", "stock": 0, ... }

// Sold out one variant
{ "id": "obsidian-black", "label": "Obsidian Black", "stock": 0 }

// Sold out one colour swatch
"colors_soldout": ["Terracotta"]
Configuration

config.js

The single configuration entry point. Located at js/config.js. All sections are explained below.

Shop Identity

shopName: "Dornori",
tagline:  "Curated lighting for modern spaces",
shopNameThe brand name shown in the page title and email subjects.
taglineSubtitle shown on the shop index page.

Currency

baseCurrency: "EUR",  // internal currency — all prices in products.json are in this

All product prices are stored in EUR. The currency selector converts them to other currencies using rates from currencies.csv. Do not change baseCurrency unless you also update all prices.

Language

defaultLanguage: "en",  // "en" | "de" | "nl" | "no"
storageKeys: {
  parentLangKey: "dornori-lang",  // key your parent site writes to localStorage
  shopLangKey:   "dornori-lang",  // key the shop reads — keep same as above
}

Feature Flags

features: {
  showLanguageSwitcher: false,  // show EN/NO/NL/DE toggle in header
  showCurrencySelector: true,   // show currency dropdown in header
}

Images

images: {
  imageExt: "webp",
  imageDir: "images/products/",
}

When a variant has no explicit image set, the shop constructs a filename using the pattern: {imageDir}{productId}_{variantId}.{imageExt}. Example: images/products/arc-floor-lamp_matte-black.webp. If that file doesn't exist the product-level image is used as fallback.

Tax

taxRate:            0.25,         // 25% VAT
taxLabel:           "VAT (25%)",
taxExemptCountries: [],           // ISO codes exempt from VAT, e.g. ["US","AU"]
businessVatExempt:  false,        // true = B2B orders are tax-free

Data File Paths

data: {
  shippingCsv:     "data/shipping.csv",
  currenciesCsv:   "data/currencies.csv",
  langUiDir:       "data/lang/ui/",
  langProductsDir: "data/lang/products/",
  productsJson:    "data/products.json",  // ← change path here if you rename it
}
Configuration

Shipping — shipping.csv

Located at data/shipping.csv. Contains global defaults and per-country overrides. The file has two sections separated by [settings] and [country_rates] headers.

Global Settings

KeyDefaultDescription
base_rate8.50Flat base shipping cost (EUR) added to every order
per_kg_rate1.20Additional cost per kg of total cart weight
free_threshold150.00Order subtotal (EUR) above which shipping is free
max_free_weight20Max weight (kg) eligible for free shipping — very heavy orders still pay
estimated_days_default3–5Default delivery estimate shown at checkout

Country Rates

Each row adds a surcharge on top of the global rates. The final shipping cost for a country is: (base_rate + country.base_eur) + (per_kg_rate + country.per_kg_eur) × weight

ColumnDescription
country_codeISO 3166-1 alpha-2 code (e.g. NL, DE, US)
country_nameDisplay name shown in the country selector
zoneInternal zone label (A–E). Not currently used in calculations.
base_eurCountry surcharge added to global base_rate
per_kg_eurCountry per-kg surcharge added to global per_kg_rate
free_threshold_overrideIf set, overrides the global free_threshold for this country
estimated_daysDelivery estimate string shown at checkout for this country
notesInternal note (customs warnings etc.). Not shown to customers.

To add a new country simply add a new row. To remove a country delete its row — it will no longer appear in the checkout country selector.

Configuration

Currencies — currencies.csv

Located at data/currencies.csv. Each row defines one currency that appears in the currency selector.

ColumnDescription
codeISO 4217 currency code (e.g. EUR, USD, GBP)
symbolSymbol prepended to amounts (e.g. €, $, kr)
nameFull currency name for the dropdown label
rate_to_eurExchange rate: 1 EUR = this many units. Keep this updated.
bufferAdditional buffer amount added to rate. Use for exchange fee cushion.
decimals2 = show cents, 0 = round to whole number (good for JPY, KRW)
localeBCP 47 locale used for number formatting (e.g. de-DE formats 1.234,56)
 
// Example: add Swiss Franc
CHF,Fr,Swiss Franc,0.9650,0.0200,2,de-CH

To remove a currency delete its row. To add one append a new row. The first currency in the file is used as the display default (EUR).

Languages

The shop supports four languages: en, de, nl, no. Each language has two files:

FileContains
data/lang/ui/en.jsonAll UI text: button labels, form fields, headings, error messages
data/lang/products/en.jsonProduct names, descriptions, category names, spec section titles

Translating UI Text

Open data/lang/ui/en.json and copy it to data/lang/ui/fr.json (for example). Translate each value while keeping keys unchanged.

Translating Product Text

Open data/lang/products/en.json. Each product ID maps to its translated name and description. Missing keys fall back to the English version.

{
  "arc-floor-lamp": {
    "name":        "Arc-Stehlampe",
    "description": "Eine Designikone für jeden Wohnraum...",
    "category":    "Stehlampen"
  }
}

Adding a New Language

1. Create data/lang/ui/fr.json and data/lang/products/fr.json
2. Add country rows with that language to shipping.csv — the shop auto-detects supported languages from it
3. Optionally set showLanguageSwitcher: true in config.js

JavaScript API

Render Functions

All public functions are available on the global Shop object. Call them after the shop scripts have loaded.

Shop.renderShop(divId, options)

Renders the full product grid with category filter into any <div>. Used on index.html.

// Minimal usage
Shop.renderShop("my-shop-div");

// With options
Shop.renderShop("my-shop-div", {
  columns:    "auto",  // "auto" (responsive) or a number like 3
  showFilter: true,    // show/hide the category filter bar
});

Shop.renderProductInfo(divId, productId)

Renders a full product info widget (gallery, variants, qty selector, add to cart) into any <div>. Used in buy-now-example.html §8.

Shop.renderProductInfo("product-info", "arc-floor-lamp");

Shop.renderCartIcon(options)

Renders the cart icon with item count badge into any element. Clicking it opens the mini-cart flyout.

Shop.renderCartIcon({
  target: "#cart-icon-slot",  // CSS selector for container
  fixed:  true,              // true = fixed position bottom-right
});

Shop.renderMiniCart(divId, options)

Renders a mini cart summary (items + totals + checkout button) into any <div>. Updates automatically when the cart changes.

Shop.renderMiniCart("sidebar-cart", {
  cartUrl: "cart.html",  // where the "Checkout" button links to
});

Shop.renderCurrencySelector(target)

Renders the currency dropdown into any element.

Shop.renderCurrencySelector("#currency-slot");

Shop.renderBackButton(target, options)

Renders a "← Back to Shop" button.

Shop.renderBackButton("#back-btn-slot", {
  href:  "index.html",  // defaults to index.html
  label: "← Back"       // optional custom label
});
JavaScript API

Cart Functions

Shop.addToCart(product, qty, variantId, color, imageOverride)

Adds a product to the cart. Dispatches a shop:cartUpdated event on success.

// Add 1 unit, no variant
Shop.addToCart(product, 1);

// Add 2 units of a specific variant
Shop.addToCart(product, 2, "matte-black");

// Add with a colour and a custom cart image
Shop.addToCart(product, 1, null, "Smoke", "https://image-url.jpg");

Shop.getCart()

Returns the current cart array from localStorage.

const cart = Shop.getCart();
// → [{ id, cartKey, name, qty, price, weight, image, selectedColor, variantId }]

Shop.removeFromCart(cartKey)

Removes an item by its cartKey. Dispatches shop:cartUpdated.

Shop.removeFromCart("arc-floor-lamp_matte-black");

Shop.updateQty(cartKey, qty)

Updates the quantity of an item. Pass qty = 0 to remove it.

Shop.updateQty("arc-floor-lamp_matte-black", 3);

Shop.calculateTotals(cart, isBusiness, countryCode)

Calculates subtotal, tax, shipping and total for a cart. Returns an object.

const { subtotal, tax, shipping, total, totalWeight, isFreeShipping }
  = Shop.calculateTotals(cart, false, "DE");

JavaScript API

Events

The shop dispatches custom DOM events on document that you can listen to for integration with other page elements.

EventWhen it fires
shop:cartUpdatedAny time the cart changes (add, remove, qty change). All cart-displaying components re-render on this event.
shop:langChangedWhen the language is switched. All rendered components rebuild with translated text.
currency:changedWhen the currency is switched. All price displays update.
 
// Example: update a custom element when the cart changes
document.addEventListener("shop:cartUpdated", () => {
  const cart  = Shop.getCart();
  const count = cart.reduce((a, i) => a + i.qty, 0);
  document.getElementById("my-count").textContent = count;
});
Customisation

Branding & CSS

All colours, fonts and spacing are defined as CSS variables at the top of css/shop.css. Edit these to restyle the entire shop without touching any other code.

:root {
  /* Brand colours */
  --c-accent:       #c8a96e;   /* gold — buttons, highlights */
  --c-accent-light: #f5ede0;   /* soft gold background */
  --c-accent-dark:  #9a7a4a;   /* dark gold text */

  /* Surface colours */
  --c-bg:           #f8f5f0;   /* page background */
  --c-surface:      #ffffff;   /* card background */
  --c-surface-2:    #f0ece4;   /* subtle alt background */

  /* Text */
  --c-text:         #1a1714;   /* primary text */
  --c-text-2:       #4a4540;   /* secondary text */
  --c-text-3:       #8a847e;   /* muted/hint text */

  /* Typography */
  --font-display:   'Cormorant Garamond', serif;
  --font-body:      'DM Sans', sans-serif;
}

The shop logo text LUMIO is hardcoded in each HTML file's <header>. To change the brand name, find and replace it in all .html files or use your editor's global find-and-replace.

Customisation

Adding a New Product

Everything is done in data/products.json. Follow these steps:

Step 1 — Add to products.json

{
  "id":       "my-new-lamp",   // unique slug — lowercase, hyphens only
  "price":    129,
  "weight":   1.5,
  "stock":    20,
  "category": "wall",
  "featured": false,
  "image":    "https://your-cdn.com/my-lamp.jpg",
  "images":   ["https://your-cdn.com/my-lamp.jpg"],
  "videos":   [],
  "url":      "product.html?id=my-new-lamp"
}

Step 2 — Add translation (optional)

Open data/lang/products/en.json and add:

"my-new-lamp": {
  "name":        "My New Lamp",
  "description": "A beautiful lamp that...",
  "category":    "Wall Lamps"
}

If you skip this step, the product's id is shown as the name and description is blank.

Step 3 — Done

Creating a New Dedicated Product Page

While product.html?id=your-id works for any product, you may want a clean URL like my-lamp.html for a featured product. Here's how:

Option A — URL parameter (simplest)

// In products.json, set the url field:
"url": "product.html?id=my-new-lamp"

The product card links to product.html?id=my-new-lamp. The product page loads that product by ID. No new files needed.

Option B — Dedicated HTML file

1. Copy product.html to my-lamp.html
2. Find this line near the top of the script section:

const prodId = params.get("id") || "arc-floor-lamp";

3. Change the fallback to your product ID:

const prodId = params.get("id") || "my-new-lamp";

4. Update the product's url field:

"url": "my-lamp.html"

The ?id= parameter still works on the dedicated page — so my-lamp.html?id=globe-pendant would show the Globe Pendant on your page.

Order Emails with Formspree

After each successful payment the shop sends an order notification to you via Formspree. The payload includes:

FieldContents
order_refUnique order reference (e.g. LM-K2B3A-XF4Z1)
customer_nameFirst + last name
customer_emailEmail address
cart_itemsLine items: qty × name @ price | weight
total_displayTotal in selected display currency
total_eurTotal in EUR
shipping_addressFull formatted address

To set up: create a free account at formspree.io, create a new form, copy the form ID into config.js:

formspree: {
  id:       "abcdefgh",
  endpoint: "https://formspree.io/f/abcdefgh",
}

Tips & Tricks

Linking directly to a product

Use product.html?id=YOUR-PRODUCT-ID anywhere — in emails, social media, other pages. The product ID must match exactly what's in products.json.

Hiding a product temporarily

To hide a product from the shop without deleting it, remove it from products.json or set "stock": 0. A stock-0 product still appears in the grid but shows "Out of Stock".

Using your own images

Replace Unsplash URLs with your own: "image": "images/my-product.jpg". Create an images/ folder in the shop root and put your files there. Recommended size: 800×800px or larger, JPEG or WebP.

Embedding the shop in another site

Call Shop.renderShop("my-div") on any page that includes the shop scripts. The shop scripts are self-contained — they don't conflict with other page code.

Forcing a specific language via URL

index.html?lang=de   ← loads in German
index.html?lang=nl   ← loads in Dutch

Forcing a specific currency via URL

index.html?currency=USD   ← loads with USD selected

Testing the checkout without PayPal

Set activeProcessor: "none" in config.js. The checkout form still works and submits the order email via Formspree, but the PayPal button is replaced with a plain "Place Order" button.
