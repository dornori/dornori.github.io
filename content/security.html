Dornori Security Policy

Security & Vulnerability Disclosure

Security Policy

Effective Date: March 31, 2026  ·  Document version 1.0

Found a vulnerability? Report it responsibly to security@dornori.com. We commit to acknowledging your report within 72 hours and providing regular status updates.

1. Our Security Commitment

Dornori is committed to protecting the security of our website, services, and hardware products, and the Personal Data of our customers — with particular care for data relating to children. We implement industry-standard technical and organisational security measures and continuously review our security posture.

2. Technical Security Measures

2.1 Encryption & Transport Security

  • All data in transit is encrypted using TLS 1.2 or higher
  • HTTP Strict Transport Security (HSTS) is enforced
  • Sensitive data at rest is encrypted using AES-256 or equivalent
  • Payment card data is handled exclusively by PCI-DSS Level 1 certified processors

2.2 Infrastructure

  • Web Application Firewall (WAF) and DDoS mitigation via Cloudflare
  • Regular vulnerability scanning and patch management
  • Access to production systems restricted by role-based access controls and multi-factor authentication
  • Separate development, staging, and production environments

2.3 Application Security

  • Input validation and output encoding applied throughout
  • CSRF protection on all state-changing requests
  • Content Security Policy (CSP) headers implemented
  • Dependency vulnerability scanning via automated tooling

2.4 Hardware Product Security

Dornori hardware products that connect to digital services are designed with security in mind. Where applicable, this includes secure boot processes, encrypted communication channels, and over-the-air (OTA) firmware update mechanisms. Product-specific security details are available in the product documentation.

3. Organisational Measures

  • Access to Personal Data is limited to personnel who require it for their role (need-to-know principle)
  • All staff with data access receive security and privacy awareness training
  • Third-party vendors are vetted for security and bound by Data Processing Agreements
  • An incident response plan is maintained and tested regularly
  • Security reviews are conducted before launching new features that process Personal Data

4. Data Breach Response

In the event of a personal data breach, we will:

  1. Contain the breach as quickly as possible
  2. Notify the relevant supervisory authority within 72 hours of becoming aware of the breach (where required under GDPR / UK GDPR)
  3. Notify affected individuals without undue delay where the breach is likely to result in high risk to their rights and freedoms
  4. Document all breaches in our internal breach register

5. Vulnerability Disclosure Policy

We encourage responsible disclosure of security vulnerabilities. If you discover a security issue affecting dornori.com or any Dornori product, please disclose it to us responsibly by following the guidelines below.

5.1 How to Report

Email: security@dornori.com

Please include:

  • A description of the vulnerability and its potential impact
  • Steps to reproduce the vulnerability
  • Any supporting material (screenshots, proof-of-concept code)
  • Your contact details for follow-up

You may encrypt sensitive disclosures using our PGP public key (available upon request at security@dornori.com).

5.2 Our Commitments to Researchers

  • Acknowledge receipt within 72 hours
  • Provide a status update within 14 days
  • Work to resolve valid vulnerabilities within 90 days
  • Notify you when the issue is resolved
  • Not pursue legal action against researchers who act in good faith per these guidelines

5.3 Scope

In scope: dornori.com and its subdomains, Dornori hardware product firmware and associated APIs, and the Dornori mobile application (if applicable).

Out of scope: Third-party services used by Dornori (report directly to those vendors), physical attacks on hardware, social engineering attacks against Dornori staff, denial-of-service attacks.

5.4 security.txt

Our machine-readable security disclosure contact is available at:

https://dornori.com/.well-known/security.txt

This file conforms to RFC 9116 and is maintained in accordance with IETF best practices for security contact disclosure.

6. No Security Guarantees

While we implement reasonable and industry-standard security measures, no system is completely secure. We cannot guarantee that our Website or services will be entirely free from security vulnerabilities or that data transmissions over the internet will always be completely secure. We will however respond promptly to any known issues.
