Dornori — Shop Component Showcase

Dornori — Component Showcase

1 — Cart Icon + Hover Dropdown

Shop.renderCartIcon()

The sticky cart icon in the header (top-right) is rendered by renderCartIcon(). Hover it to reveal a dropdown showing live cart contents, subtotal, shipping status, and a checkout button. The badge updates reactively via shop:cartUpdated events. Add any product below and watch the badge and dropdown update instantly.

2 — Product Gallery + Category Filter

Shop.renderShop()

renderShop() reads /data/products.json, auto-generates category filter buttons (All / lamp / kit / parts / accessory), renders all product cards with variant selectors, quantity controls, per-card "Add to Cart", badge overlays (Featured, Best Seller, % OFF), and related-product strips. Clicking a variant button swaps the card image and recalculates price.

3 — Product Info Panel (PDP)

Shop.renderProductInfo()

renderProductInfo() renders a full product detail widget: image gallery with thumbnails, variant selector, quantity control, pricing (with discount), stock indicator, weight/dimensions, Add to Cart, View Cart button, specs/details tab bar, and a related-products strip below. Selecting a variant swaps the main image and price. Shown here for pre-assembled.

4 — Mini Cart

Shop.renderMiniCart()

renderMiniCart() renders a persistent sidebar-style cart summary: item list with images, variant label, quantity × price, per-item weight, subtotal, shipping (FREE above threshold), total weight, grand total, remove buttons, and a Proceed to Checkout CTA. Reacts to shop:cartUpdated in real time — add items from Section 2 or 3 to see it populate.

5 — Product Embeds with Feature Flags

data-shop-products="[id]" [flags]

Any element with data-shop-products="[id]" gets a product card injected. By default, the card shows only the product image, name, price, and "Add to Cart" button. Add feature flags to enable variants selection (data-variants) and related products (data-related) strips. Useful for embedding individual products inside editorial content.

Default (Minimal)

<div data-shop-products="ufo-spaceblue"></div>

With Variants (data-variants)

<div data-shop-products="pre-assembled" data-variants></div>

With Related Products (data-related)

<div data-shop-products="electronics-bundle" data-related></div>

All Flags Combined

<div data-shop-products="pre-assembled" data-variants data-related></div>

6 — Currency Selector

Shop.renderCurrencySelector()

renderCurrencySelector() renders a dropdown populated from the Currency module. Selecting a currency fires currency:changed and all prices across every component (cards, PDP, mini cart, hover panel) re-render in the chosen currency. A second instance is shown here inline; the primary lives in the sticky header.

Report — What to Improve & Add

Improvements

  • Image zoom / magnifier: The product.html page has a cursor-crosshair magnifier but it is not wired into renderProductInfo(). Bringing it into the widget would match Amazon/Apple UX.
  • Skeleton loading screens: All four widgets flash blank → populated. Add CSS skeleton shimmer placeholders that swap out when data resolves, preventing layout shift.
  • Stock countdowns: products.json has stock: 99 everywhere. Real stock numbers + a "Only 3 left!" low-stock label (Temu/Amazon style) create urgency.
  • Discount expiry / countdown timer: The discount badge shows "15% OFF" with no context. A "Deal ends in 2h 14m" timer (Apple promo style) dramatically increases conversion.
  • Variant image thumbnails: Variant selector buttons are text-only. Showing a small swatch image per variant (like Apple's colour selector) gives shoppers an instant visual preview.
  • Related strip position: "You may also need" currently renders inside the card body, pushing button below fold on long names. Move it below the ATC button or into a drawer.
  • Cart hover panel on mobile: The dropdown is desktop-only. On touch devices replace it with a slide-up drawer triggered by the cart icon tap.
  • SEO / structured data: renderProductInfo() injects no application/ld+json Product schema. Adding it costs nothing and enables Google Shopping rich results.

New Functions to Add

  • renderWishlist(divId): Heart-icon toggle on every card, a dedicated wishlist page. Storage in localStorage. Share-link generation. Standard on Amazon, Apple, Temu.
  • renderRecentlyViewed(divId): Track product IDs visited in session storage; render a horizontal scroll strip. Amazon's most effective retention tool.
  • renderProductSearch(divId): Client-side fuzzy search over products.json names + categories with live results dropdown. No server required.
  • renderBundleBuilder(divId, ids[]): "Build your bundle" widget — select compatible items (e.g. Kit + Electronics + Batteries), show combined price with bundle discount. Apple Studio bundle style.
  • renderCompare(divId, ids[]): Side-by-side spec table for 2-3 products. Common on electronics stores; reduces support queries and returns.
  • renderReviews(divId, productId): Star rating histogram, review cards with verified badge, sort by rating/date. Rating data is already in products.json; just needs display + write API.
  • renderStockAlert(productId): "Notify me when back in stock" email capture form for out-of-stock variants. Temu/Amazon staple for recovering lost sales.
  • renderCheckoutProgress(): A sticky 3-step breadcrumb (Cart → Shipping → Payment) so users always know where they are. Currently the cart page has no progress indicator.
  • renderCrossSell(divId, cartItems): Post-add-to-cart "Frequently bought together" row generated from the related arrays in products.json. High-ROI upsell.
  • renderInstallmentCalc(price): "Pay in 3 × €XX" label below price (Klarna / PayPal Pay Later style). Increases AOV by reducing perceived price barrier.
